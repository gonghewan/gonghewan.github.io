Linux基础

系统分区

- 主分区最多有四个，其中可至多包含一个扩展分区，扩展分区用于划分逻辑分区

  • 格式化（逻辑格式化），指将一个分区进行划分为多个块，在linux中格式化将分区划分为等大小的块（block），每块4kb，当一个block被一个文件占用后将不可在被其他文件更改。当存储一个文件时，有一个表（i-node list）来记录文件和其对应的位置，每个文件都有其各自的编号（i-节点号/i-node num），其对应着文件存储的block集合，当查找文件时按照这个表来查询文件都存储在哪些块中。

  • 为每个分区自动生成硬件设备文件名

    如：/dev/sda1 sata接口1分区

    逻辑分区从5开始

  • 挂载（linux分配挂载点（即win的盘符））

    必需分区：

    • 根分区
    • swap分区（虚拟内存，某些情况下用该分区作为补充内存）

    推荐分区：

    • /boot(启动分区，用于解决根分区满了的情况下启动失败的问题)

    Linux实际分区情况：

    即当向home写入时，实际是向/dev/sda2写入，向boot写入时实际是向/dev/sda1写入，向etc写入时实际是向/dev/sda3写入

linux：数据写入硬盘必须经过分区->格式化->为分区设置其硬件设备文件名（linux自动完成）->分配挂载点->写入

windows：数据写入硬盘必须经过分区->格式化->分配盘符->写入

Linux常用命令

文件处理命令

- ls +参数 +文件名

  • 命令所在路径：/bin/ls

  • -a 查看所有文件，包括隐藏文件 –all

  • -l 显示文件的详细属性 –long

    • 使用权限 引用系数 所有者 所有组 大小

      u g o

      所有者 所属组 其他人

      -rw-r—r–

      lrw-r–r–

      drw-r–r–

      首字符-表示普通文件

      ​ d表示目录

      ​ l表示软链接

  • -d 查看目录属性 ls -ld

  • -h 人性化显示，常用于选择合适单位显示文件大小 –human

  • -i 查看inode（索引号）

  • mkdir + path1（新文件路径）+ path2（新文件路径）

    • -p 递归新建多层子目录

  • pwd （显示当前文件目录）

  • rmdir + path 删除空目录

  • cp +src1+src2 +des (复制同时可以实现改名)

    • -rp
    • -r 递归复制目录
    • -p 保留文件属性

  • mv +src1+src2+des（剪切同时可以实现改名）

  • rm 删除

    • -r 删除目录
    • -f 强制删除

  • ln（link）

    • ln -s src des//软链接

    • ln src des//硬链接，区别于cp -p，可以同时更新两个文件

      当删除了原文件时，软连接文件会消失，硬链接还可以正常访问，识别一个文件是软链接文件：ls -l显示lrwxrwxrwx，识别一个文件是一个硬链接：ls -i查看inode，两个文件同时指向一个inode，当删除原文件时inode的指针会减一，硬链接的文件必须在同一个分区中，且不能对目录操作。

  • 文件搜索命令

    • find

      find 搜索范围 匹配条件

      • find /etc -name init

        /etc 下文件名为init的文件

      • find /etc -name init\

        /etc下文件名包含init的文件

      • find /etc -name init???

        /etc下文件名为init开头后加三个字符的文件

      • find /etc -iname init???

        不区分大小写

      • find / size +204800

        在根目录下查找大于100MB的文件

        1个数据块 512字节

      • find /home -user shenchao

        查找所属者为shenchao的文件

      • find /etc -cmin -5

        查找5分钟内被修改过属性的文件和目录

        -amin访问时间 -cmin文件属性 -mmin文件内容

      • find /etc -size +163840 -a -size -204800

        在根目录下查找大于80MB小于100MB的文件

        -a 两个条件同时满足

        -o 两个条件满足一个

      • find /etc -name init -exec ls -l {} ;

        在/etc下查找init文件并显示其详细信息

        -exec/-ok +命令+{}; 对搜索结果执行操作

      • -type 根据文件类型查找

        f 文件 d目录 l链接文件

      • -inum 根据i节点查找

        ls -i查出inum

        find . -inum xxx -exec rm {} ;

        还可用于查找一个文件的硬链接

        ls -i 查出inum后，在该分区下查找find . -inum xxx可找到所有指向该节点的文件

  • 影子文件/etc/shadow

    存储加密真正的密码后得到的密文

  • passwd与shadow文件之间的联系

    shadow文件中用户密码的部分是*或者!!时代表无密码，不能登录

    当给用户密码前添加!后代表禁用该用户

    passwd文件中密码部分是x的情况下，当用户登陆时系统会去查询shadow文件，进行密码对比，以root身份可将x去掉表示该用户无密码，即可免去登陆验证
